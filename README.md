# Quick Build

## Next.js + Clerk + Convex Template

### Build your projects faster without worrying about the boring stuff

---

This template includes everything you need to get your project up and running without hassling with authentication, databases, and connecting them all to your Next.js app. It comes pre-configured with shadcn/ui for beautiful components, Clerk for authentication, and Convex for your backend and database.

### What you'll need:

- A [Clerk account](https://clerk.com) (free tier available)
- A [Convex account](https://convex.dev) (free tier available)
- Node.js installed on your machine

### How to use it:

1. Run `git clone https://github.com/josephheinz/Quick-Build.git`
2. Run either `pnpm install` or `npm install` to install all of the packages
3. Follow the setup instructions below based on whether you want to develop locally or in the cloud

---

## Setup Guide

### Option 1: Local Development (Recommended for Getting Started)

Local development runs Convex on your machine without needing to create a Convex account initially. This is faster for development and doesn't count against usage quotas.

1. **Initialize Convex locally:**

   ```bash
   npx convex dev
   ```

   When prompted, choose **"Start a local backend"** option

2. **Create a `.env.local` file** with the following variables:

   ```env
   # Convex (automatically generated by npx convex dev)
   CONVEX_DEPLOYMENT=anonymous:anonymous-nrcc-template
   NEXT_PUBLIC_CONVEX_URL=http://127.0.0.1:3210

   # Clerk (get these from your Clerk dashboard)
   NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_****
   CLERK_SECRET_KEY=sk_test_****
   ```

3. **Set up Clerk:**
   - Go to [Clerk Dashboard](https://dashboard.clerk.com)
   - Create a new application (or use an existing one)
   - Navigate to **API Keys** in the sidebar
   - Copy your **Publishable Key** and **Secret Key** and paste them into your `.env.local` file

4. **Configure Clerk JWT template:**
   - In your Clerk dashboard, go to **JWT Templates**
   - Create a new template and select **Convex**
   - Copy the **Issuer URL** (format: `https://verb-noun-00.clerk.accounts.dev`)
   - Since you're running locally, you can skip adding this to Convex for now
   - Once you switch to cloud deployment, you'll need to add this as `CLERK_JWT_ISSUER_DOMAIN` in your Convex dashboard

5. **Run your app:**
   ```bash
   npm run dev
   ```

**Note:** With local development, your Convex backend runs as a subprocess. You'll need `npx convex dev` running to use the database.

### Option 2: Cloud Development

Cloud development connects to Convex's hosted backend, which is recommended for production and team collaboration.

1. **Initialize Convex with cloud deployment:**

   ```bash
   npx convex dev
   ```

   When prompted:
   - Choose **"Configure a new Convex project"**
   - Log in to your Convex account (or create one)
   - Create a new project

   This will automatically create a `.env.local` file with your deployment URL.

2. **Your `.env.local` should look like this:**

   ```env
   # Convex (automatically generated)
   CONVEX_DEPLOYMENT=dev:adjective-animal-123
   NEXT_PUBLIC_CONVEX_URL=https://adjective-animal-123.convex.cloud

   # Clerk (add these manually)
   NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_****
   CLERK_SECRET_KEY=sk_test_****
   ```

3. **Set up Clerk** (same as local development):
   - Go to [Clerk Dashboard](https://dashboard.clerk.com)
   - Create a new application
   - Navigate to **API Keys**
   - Copy your **Publishable Key** and **Secret Key** into `.env.local`

4. **Configure Clerk JWT template and connect to Convex:**
   - In Clerk dashboard, go to **JWT Templates** → Create new → Select **Convex**
   - Copy the **Issuer URL** (format: `https://verb-noun-00.clerk.accounts.dev`)
   - Go to your [Convex Dashboard](https://dashboard.convex.dev)
   - Navigate to your project → **Settings** → **Environment Variables**
   - Add a new variable:
     ```
     CLERK_JWT_ISSUER_DOMAIN=https://verb-noun-00.clerk.accounts.dev
     ```
   - **Important:** Run `npx convex dev` or `npx convex deploy` after adding this variable to sync the configuration

5. **Run your app:**
   ```bash
   npm run dev
   ```

## Switching from Local to Cloud

If you started with local development and want to switch to cloud:

1. Run `npx convex login` to authenticate
2. Run `npx convex dev` and select **"Configure a new Convex project"**
3. Your `.env.local` will be updated with your cloud deployment URL
4. Follow step 4 from the Cloud Development section to add `CLERK_JWT_ISSUER_DOMAIN` to your Convex dashboard
5. Update Clerk environment variables if needed (production vs development keys)

## Environment Variables Reference

| Variable                            | Where to find it                                  | Required for                           |
| ----------------------------------- | ------------------------------------------------- | -------------------------------------- |
| `CONVEX_DEPLOYMENT`                 | Auto-generated by `npx convex dev`                | Local/Cloud                            |
| `NEXT_PUBLIC_CONVEX_URL`            | Auto-generated by `npx convex dev`                | Local/Cloud                            |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk Dashboard → API Keys                        | Both                                   |
| `CLERK_SECRET_KEY`                  | Clerk Dashboard → API Keys                        | Both                                   |
| `CLERK_JWT_ISSUER_DOMAIN`           | Clerk Dashboard → JWT Templates → Convex template | Cloud only (added to Convex dashboard) |

## Troubleshooting

**Authentication not working?**

- Make sure you've created the Convex JWT template in Clerk
- Verify `CLERK_JWT_ISSUER_DOMAIN` is set in your Convex dashboard (for cloud deployments)
- Run `npx convex dev` or `npx convex deploy` to sync the auth configuration

**Can't connect to Convex?**

- For local development, ensure `npx convex dev` is running
- For cloud development, check that your `NEXT_PUBLIC_CONVEX_URL` matches your dashboard

**Wrong deployment keys?**

- Development keys start with `pk_test_` and `sk_test_`
- Production keys start with `pk_live_` and `sk_live_`
- Make sure you're using the right environment keys for your deployment
